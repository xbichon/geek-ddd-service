## 1. 功能列表
- 新增菜单；
- 编辑菜单；
- 删除菜单；
- 查看菜单（单个）；
- 获取菜单树；

## 2. 实体设计
| 字段名        | 类型 | 描述              |
|------------| --- |-----------------|
| id         | Long | 主键ID            |
| name       | String | 菜单名称            |
| parentId   | Long | 父菜单ID           |
| type       | Integer | 菜单类型（1：目录，2：菜单） |
| url        | String | 菜单URL           |
| icon       | String | 菜单图标            |
| remark     | String | 备注              |
| level      | Integer | 层级              |
| sortOrder  | Integer | 排序号             |
| createTime | LocalDateTime | 创建时间            |
| updateTime | LocalDateTime | 更新时间            |
| version    | Integer | 版本号             |

## 3. 新增菜单
### 3.1 需求描述
- 新增菜单；
- 支持新增子菜单；
- 支持新增同级菜单；

### 3.2 业务约束
- 菜单名称同级下不能重复,不能为空；
- 菜单层级不能超过5级
- 菜单类型不能为空；
- 菜单类型为目录时，url必须为空；
- 菜单类型为菜单时，url不能为空；

### 3.3 业务流程
1. 用户在前端界面填写菜单相关信息后，点击 “添加菜单” 按钮；
2. 后端接收到请求后，进行进行业务处理；
   1. 参数验证：
   2. 业务校验：
      - 校验菜单名称（同一级别下，不能重复）；
      - 校验父级ID（必须合法、存在）；
   3. 计算最大排序号；
   4. 创建菜单对象；
      - 计算层级；
         > 如果没有父级，层级为1；
         > 如果有父级，层级为父级层级+1；
   5. 保存菜单信息；
   6. 返回结果；

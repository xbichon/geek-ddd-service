## 🛠️ 技术栈

- **框架**: Spring Boot 3.x
- **架构**: DDD + CQRS
- **数据库**: MySQL
- **ORM**: JPA + JOOQ
- **数据库迁移**: Liquibase
- **构建工具**: Maven
- **日志**: Logback
- **安全**: Spring Security + JWT

## 📖 核心概念

### DDD (领域驱动设计)
- **聚合根**: 业务实体的入口点，维护一致性边界
- **值对象**: 不可变的对象，包含业务验证逻辑
- **领域服务**: 处理跨聚合的业务逻辑
- **仓储**: 数据访问的抽象接口

### CQRS (命令查询职责分离)
- **命令**: 表示写操作意图，包含业务验证
- **查询**: 专门用于数据查询和展示
- **命令处理器**: 处理具体的业务逻辑
- **查询服务**: 处理复杂的数据查询

## 📝 开发流程

1. **需求分析** → 明确业务需求和边界
2. **领域建模** → 识别聚合根、实体、值对象
3. **数据库设计** → 创建 Liquibase 变更文件
4. **代码生成** → 生成 JOOQ 代码
5. **领域实现** → 实现聚合根和值对象
6. **应用服务** → 实现命令和查询
7. **接口开发** → 实现 REST API
8. **测试验证** → 单元测试和集成测试
